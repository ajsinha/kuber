About Regions

Regions provide logical isolation for cache entries. Each region:

  • Maps to a separate MongoDB collection
  • Has its own statistics and metrics
  • Can be independently purged or deleted
  • Supports access control policies
Naming Conventions
  • users - User profile data
  • sessions - Session data with TTL
  • products - Product catalog
  • cache-v1 - Versioned caches
The default region is captive and cannot be deleted. Use it for general-purpose caching or create specific regions for better organization.
