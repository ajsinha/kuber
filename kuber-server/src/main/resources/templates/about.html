Kuber Distributed Cache

A high-performance, Redis-compatible distributed caching solution with pluggable persistence (RocksDB, Aerospike, MongoDB, and more), JSON document support, and enterprise-grade features.

Version 2.6.0 Production Ready Patent Pending

The name Kuber derives from Kubera (Sanskrit: कुबेर), the revered deity in Hindu mythology who is the divine keeper, guardian, and treasurer of wealth. As the celestial custodian of riches and precious resources, Kubera watches over and safeguards the world's treasures — just as the Kuber cache system guards and preserves your most valuable data assets with unwavering reliability.

High Performance

In-memory caching powered by Caffeine with sub-millisecond response times. Handles millions of operations per second.

Redis Compatible

Use your existing Redis clients and tools. Supports standard Redis protocol commands plus powerful extensions.

Multi-Backend Persistence

Choose your persistence backend: RocksDB, LMDB, MongoDB, SQLite, PostgreSQL, or Aerospike. Each region gets its own dedicated database instance for maximum concurrency.

JSON Document Support

Store, query, and manipulate JSON documents with JSONPath expressions. Attribute mapping transforms field names on storage.

Region Isolation

Each region gets its own dedicated database instance (RocksDB/LMDB/SQLite) for better concurrency, isolation, and performance. Independent compaction per region.

Replication Support

Primary/Secondary replication via ZooKeeper for high availability. Automatic failover and data synchronization.

Autoload

Bulk data import from CSV, TXT, and JSON files. Drop files in inbox folder and Kuber automatically loads them into cache with configurable TTL and regions.

Event Publishing

Stream cache events to Kafka, RabbitMQ, IBM MQ, ActiveMQ, or files. Hot-refresh bindings at runtime. Publish query results and entire regions on demand.

Enhanced in v2.6.0
Request/Response Messaging

Access cache via message brokers (Kafka, RabbitMQ, IBM MQ, ActiveMQ). Async processing with backpressure control, hot-reload, and built-in test console.

Enhanced in v2.6.0
C# / .NET Client

Complete .NET client library with async/await support. Redis protocol, REST API, and messaging patterns.

NEW in v1.7.1
Synchronous Data Consistency

Sequential startup compaction and data loading for guaranteed data integrity. Disk writes complete before key index updates.

Updated in v1.5.0
Monitoring & Live Logs

Real-time monitoring dashboard with charts for CPU, memory, cache operations, and hit rates. Live log viewer with level filtering, text search, and auto-scroll.

v2.2.0
API Key Authentication

Secure programmatic access with API keys. Multiple authentication methods: X-API-Key header, Bearer token, or query parameter.

Version History

v2.6.0 Current - Expired Event Payload Enhancement & Admin UI Fix

February 2026

  • Expired Events with Payload: When a cache entry expires due to TTL, the published expired event now includes the full value in the payload field — mimicking the structure of inserted and updated events for downstream consumers
  • Immediate Caffeine Eviction Publishing: The Caffeine removal listener now publishes expired events immediately when entries are evicted from memory, with full value payload — no longer waits for the periodic cleanup task
  • Disk-Only Value Capture: Cleanup task reads values from persistence for cold (disk-only) entries before deletion, so all expired events carry the payload
  • Deduplication: Composite-key tracking prevents double-publishing between Caffeine listener and cleanup task
  • Admin UI Bootstrap Fix: Fixed missing Bootstrap JS on 8 admin pages and 29 help pages — the "admin" dropdown link in the top-right navbar now works on all pages
v2.5.0 Publish As Events, Autoload Events, Messaging Test Console & Serialization Fix

February 2026

  • Publish As Events — Query Page: "Publish As Events" button on /cache/query publishes all query result keys as queryresult events to configured brokers for the queried region
  • Publish As Events — Cache Browser: Same button on /cache?region=… publishes all displayed entries to brokers
  • Publish As Events — Region Detail: Button on /regions/{name} publishes the entire region's entries as events via POST /api/publish/region/{region}
  • Query Result Event Type: New QUERY_RESULT (queryresult) action type in CachePublishingEvent — structured like an INSERT event with key and payload
  • Autoload Event Publishing: When a region has event publishing configured, autoload now publishes autoload_start and autoload_end events to all configured brokers — payload includes file name, region, records loaded, errors, duration, and status
  • Messaging Test Console: New "Test Request" section on /admin/messaging with channel/topic selection, 20 pre-built example templates covering all operations, JSON editor, and one-click publish to any connected request topic
  • Instant Serialization Fix: Fixed CachePublishingEvent JSON serialization — registered JavaTimeModule on the internal ObjectMapper so java.time.Instant timestamps serialize as ISO-8601 strings instead of failing
v2.4.0 Event Publishing Overhaul & Internal Event Bus

February 2026

  • Hot-Refresh Event Publishing: Runtime config changes (add/enable/disable region, broker toggle, raw JSON save) now immediately refresh all publisher bindings without server restart
  • Complete Event Coverage: All 14 write paths now fire events to brokers — added publishing for jsonSet, jsonUpdate, jsonRemoveAttributes, jsonDelete, expire, persist, rename, purgeRegion, and cleanupExpiredEntries
  • Expired Event Type: New EXPIRED action type in CachePublishingEvent — TTL-expired entries now publish expiration events to configured brokers
  • Internal Event Bus Stats: Event Publishing admin page now shows dual stat panels for Internal Event Bus (total events, active channels, pub/sub subscribers, listeners) and Broker Publishing Queue (queue depth, active threads, errors/dropped)
  • Per-Region Published Counts: Region table on Event Publishing page displays per-region published message count with badge indicators
  • Info-Level Publish Logging: All 5 publisher implementations (Kafka, ActiveMQ, RabbitMQ, IBM MQ, File) now log successful publishes at INFO level instead of DEBUG
v2.2.0 Monitoring, Live Logs & UI Enhancements

February 2026

  • Real-Time Monitoring Dashboard: Live system metrics at /admin/monitoring with auto-refresh charts for CPU, heap, threads, cache operations, and hit/miss ratios
  • Live Log Viewer: Terminal-style log viewer at /admin/logs with level filtering, text search, auto-scroll, and log download
  • Log Buffer Service: In-memory ring buffer capturing 2000 log entries with incremental polling and custom Logback appender
  • Unified Table Component: All data tables now feature client-side pagination, column sorting, text search, and configurable page sizes via reusable KuberTable component
  • Dark / Light Theme Toggle: One-click theme switching in the navbar with localStorage persistence and OS preference detection; comprehensive dark mode across all pages including login, monitoring charts, and admin panels
  • Regions Table View: Regions page converted from cards to a sortable, searchable table with capacity bars
  • System Overview API: New /api/monitoring/overview endpoint aggregating JVM, OS, threads, GC, and cache stats in a single call
