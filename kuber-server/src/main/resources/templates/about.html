Kuber Distributed Cache

A high-performance, Redis-compatible distributed caching solution with pluggable persistence (RocksDB default), JSON document support, and enterprise-grade features.

Version 1.8.1 Production Ready Patent Pending
High Performance

In-memory caching powered by Caffeine with sub-millisecond response times. Handles millions of operations per second.

Redis Compatible

Use your existing Redis clients and tools. Supports standard Redis protocol commands plus powerful extensions.

Multi-Backend Persistence

Choose your persistence backend: RocksDB, LMDB, MongoDB, SQLite, or PostgreSQL. Each region gets its own dedicated database instance for maximum concurrency.

JSON Document Support

Store, query, and manipulate JSON documents with JSONPath expressions. Attribute mapping transforms field names on storage.

Region Isolation

Each region gets its own dedicated database instance (RocksDB/LMDB/SQLite) for better concurrency, isolation, and performance. Independent compaction per region.

Replication Support

Primary/Secondary replication via ZooKeeper for high availability. Automatic failover and data synchronization.

Autoload

Bulk data import from CSV, TXT, and JSON files. Drop files in inbox folder and Kuber automatically loads them into cache with configurable TTL and regions.

Event Publishing

Stream cache events to Kafka, RabbitMQ, IBM MQ, ActiveMQ, or files. Multi-destination support with centralized broker configuration.

v1.6.5
Request/Response Messaging

Access cache via message brokers (Kafka, RabbitMQ, IBM MQ, ActiveMQ). Async processing with backpressure control and hot-reload configuration.

NEW in v1.7.1
C# / .NET Client

Complete .NET client library with async/await support. Redis protocol, REST API, and messaging patterns.

NEW in v1.7.1
Synchronous Data Consistency

Sequential startup compaction and data loading for guaranteed data integrity. Disk writes complete before key index updates.

Updated in v1.5.0
API Key Authentication

Secure programmatic access with API keys. Multiple authentication methods: X-API-Key header, Bearer token, or query parameter.

Version History

v1.8.1 Current - Secondary Indexing & Hybrid Search Architecture

January 2026

  • Secondary Indexing: Hash and BTree indexes for O(1) and O(log n) lookups instead of full table scans
  • Hybrid Storage: In-memory indexes for queries + RocksDB persistence for durability
  • Index Configuration: Define indexes per region in config/index.yaml
  • Index Management UI: Admin page at /admin/indexes for creating, viewing, rebuilding indexes
  • Index REST API: Full CRUD operations at /api/v1/regions/{region}/indexes
  • Query Optimizer: Automatically selects best index for queries (intersection for AND logic)
  • Hash Index: O(1) lookups for equality queries (status, city, category)
  • BTree Index: O(log n) range queries (age > 30, date BETWEEN, price < 100)
  • Composite Indexes: Multi-field indexes for common query patterns
  • Auto Index Maintenance: Indexes updated automatically on INSERT/UPDATE/DELETE
  • Index Rebuild: On-demand or startup rebuild with parallel processing
  • Performance: 100-1000x speedup for indexed queries vs full scan
  • Parallel JSON Search: Multi-threaded search with up to 8x speedup on large datasets
  • Parallel Pattern Search: Two-phase parallel processing for key pattern searches
  • Search Statistics: Monitor index hit rates, query performance at /api/v1/indexes/stats
  • Hot Reload: Index configuration changes detected and applied without restart
v1.7.7 Security, SSL/TLS & Enhanced Search APIs

December 2025

  • Security & RBAC: Enterprise-grade role-based access control with region-level permissions
  • SSL/TLS Configuration: Comprehensive guide for secure server and client connections
  • Spring Boot SSL: Full configuration options via properties, YAML, command line, environment variables, and Java system properties
  • PEM File Support: Direct PEM certificate/key configuration for cloud-native deployments (Spring Boot 3.x)
  • SSL Bundles: Reusable SSL material configuration (Spring Boot 3.1+)
  • JSEARCH IN Clause: Match multiple values per attribute with [v1|v2|v3] syntax
  • Generic Update API: Unified SET/UPDATE endpoint with intelligent JSON merging (JUPDATE)
  • Multi-Key Search: Search by single key or list of keys in one request
  • Multi-Pattern Search: Search using multiple regex patterns for keys (OR logic)
  • Advanced JSON Search: Flexible attribute matching with AND logic across all criteria
  • Comparison Operators: Support for gt, gte, lt, lte, ne, eq, regex operators in JSON search
  • API Key in Body: Pass apiKey directly in JSON request body for POST requests
  • Smart Merge: JSON updates preserve existing fields, add new ones, and deep merge nested objects
  • Mutual TLS (mTLS): Client certificate authentication with trust store configuration
  • Client Library Updates: All 3 clients (Java, Python, C#) updated with jsonSearchIn() and SSL support
  • Demo Code Updates: Python and Java JSON demos updated with IN clause examples
v1.7.6 Prometheus Monitoring & Dark Blue Theme
  • Prometheus Integration: Native metrics export at /actuator/prometheus endpoint
  • 20+ Metrics: Cache operations, hit rates, evictions, memory usage, per-region stats
  • Grafana Ready: Pre-built PromQL queries for dashboards and alerting
  • Dark Blue Theme: Professional enterprise color scheme (#0079c1 primary)
  • PrometheusMetricsService: Scheduled metrics updates with configurable intervals
  • Alert Rules: Example Prometheus alert configurations included
v1.7.5 Warm Objects & Eviction Protection
  • Warm Objects: Protect frequently accessed keys from eviction
  • Access Tracking: Track access counts and last access time per key
  • Configurable Thresholds: Set min access count and time window for warm status
  • Per-Region Config: Enable/disable warm objects per region
  • Eviction Coordination: WarmObjectService coordinates with eviction to protect hot data
  • Admin Visibility: View warm object statistics in admin dashboard
