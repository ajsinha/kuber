Kuber Distributed Cache

A high-performance, Redis-compatible distributed caching solution with pluggable persistence (RocksDB default), JSON document support, and enterprise-grade features.

Version 1.1.4 Production Ready Patent Pending
High Performance

In-memory caching powered by Caffeine with sub-millisecond response times. Handles millions of operations per second.

Redis Compatible

Use your existing Redis clients and tools. Supports standard Redis protocol commands plus powerful extensions.

Multi-Backend Persistence

Choose your persistence backend: MongoDB, SQLite, PostgreSQL, or RocksDB. Data survives restarts with configurable sync modes.

JSON Document Support

Store, query, and manipulate JSON documents with JSONPath expressions. Attribute mapping transforms field names on storage.

Region Partitioning

Organize cache data into logical regions with independent namespaces. Regions are auto-created on first use - just store data and regions appear automatically.

Replication Support

Primary/Secondary replication via ZooKeeper for high availability. Automatic failover and data synchronization.

Autoload

Bulk data import from CSV and JSON files. Drop files in inbox folder and Kuber automatically loads them into cache with configurable TTL and regions.

Architecture

System Components
  • Cache Service - Core caching engine with Caffeine
  • Redis Protocol Server - Apache MINA-based protocol handler
  • REST API - Spring Boot REST endpoints
  • Web UI - Thymeleaf + Bootstrap management console
  • Persistence Layer - MongoDB repository
  • Replication Manager - ZooKeeper coordination
  • Autoload Service - CSV/JSON bulk import
Technology Stack
Runtime Java 17+, Spring Boot 3.x
In-Memory Cache Caffeine (Google)
Protocol Server Apache MINA 2.x
Persistence MongoDB 4.x+
Coordination Apache ZooKeeper (optional)
Security Spring Security 6.x
JSON Processing Jackson, JsonPath
Web UI Thymeleaf, Bootstrap 5

Supported Operations

String Operations
  • GET, SET, SETNX, SETEX
  • MGET, MSET
  • INCR, DECR, INCRBY, DECRBY
  • APPEND, STRLEN, GETSET
Key Operations
  • KEYS, DEL, EXISTS
  • TYPE, RENAME
  • EXPIRE, TTL, PERSIST
  • KSEARCH (regex search)
Hash Operations
  • HGET, HSET, HSETNX
  • HMGET, HMSET
  • HGETALL, HKEYS, HVALS
  • HDEL, HEXISTS, HLEN
JSON Operations
  • JSET, JGET
  • JSEARCH (query documents)
  • JDEL (delete path)
  • JSONPath expressions
Region Operations
  • REGIONS (list all)
  • RCREATE, RDROP
  • RPURGE, RINFO
  • RSELECT (switch region)
Server Operations
  • PING, INFO, DBSIZE
  • FLUSHDB, FLUSHALL
  • AUTH, QUIT
  • TIME
