Overview

Welcome to Kuber Distributed Cache

Kuber is a high-performance, Redis-compatible distributed caching system with:

  • Request/Response Messaging (v1.7.1) - Access cache via Kafka, RabbitMQ, IBM MQ, ActiveMQ with broker controls
  • C# / .NET Client (v1.7.1) - Complete .NET client with async/await, Redis protocol, REST API, and messaging
  • Dual Memory Management (v1.7.9) - Memory pressure eviction (heap) + count-based limits (20% or 10K per region)
  • Backup & Restore (v1.5.0) - Automatic periodic backup and auto-restore for RocksDB/LMDB
  • Async Writes (v1.3.10) - Memory updated first, disk write in background for 10-100x faster individual writes
  • Batch Autoload (v1.3.9) - Bulk data import with configurable batch size for 10-50x faster loading
  • Hybrid Memory Architecture (v1.2.2) - All keys in memory, values can overflow to disk (Aerospike-like)
  • Multi-Backend Persistence - Choose from RocksDB, LMDB, MongoDB, PostgreSQL, SQLite, or in-memory
  • JSON Document Support - Store and query complex JSON with JSONPath
  • Region Partitioning - Logical namespaces with auto-creation on first use
  • Redis Compatibility - Use existing Redis clients and tools
  • High Performance - O(1) key existence checks via KeyIndex, sub-millisecond operations
  • Replication - Primary/Secondary architecture with ZooKeeper coordination
  • Autoload - Bulk data import from CSV, TXT, and JSON files
  • Event Publishing (v1.3.2) - Stream events to Kafka, RabbitMQ, IBM MQ, ActiveMQ, or files
  • API Key Authentication - Secure programmatic access with multiple auth methods

Key Features

  • Store complex JSON documents with nested objects and arrays
  • Query specific paths using JSONPath syntax ($.field.subfield)
  • Update individual fields without replacing entire documents
  • Search by JSON attributes with comparison operators
  • Attribute mapping transforms field names automatically on storage

  • RocksDB - Embedded key-value store with separate database per region for concurrent access
  • LMDB - Lightning Memory-Mapped Database with zero-copy reads and ACID transactions
  • MongoDB - Document database with rich querying capabilities
  • PostgreSQL - Enterprise-grade relational database
  • SQLite - Lightweight embedded database with separate file per region
  • Memory - Pure in-memory for testing or ephemeral caching

RocksDB, LMDB, and SQLite use region isolation - each region has its own database instance for better concurrency and performance.

  • KEYS - Glob-style pattern matching (user:*)
  • KSEARCH - Regex key search with full value return
  • JSEARCH - Query by JSON attribute values
  • Generic Search API - Unified search with field projection
  • Comparison operators: =, !=, >, <, >=, <=, ~= (regex)

  • Apache Kafka - High-throughput streaming with auto topic creation
  • RabbitMQ - AMQP messaging with flexible exchange routing
  • IBM MQ - Enterprise messaging with SSL/TLS support
  • Apache ActiveMQ - JMS messaging with queue/topic support
  • File System - JSON Lines output with auto-rotation
  • Centralized broker definitions with multi-destination support
  • Non-blocking async publishing with configurable thread pool
