DEL - Delete Keys

Delete one or more keys. Returns the number of keys deleted.

  • Redis CLI
  • Python
  • Java
  • C#
# Delete single key
DEL user:1001
# Returns: 1 (if existed)

# Delete multiple keys
DEL user:1001 user:1002 user:1003
# Returns: 3 (number deleted)

# Delete non-existent key
DEL nonexistent
# Returns: 0
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Delete single key
    deleted = client.delete('user:1001')
    print(f"Deleted: {deleted}")  # 1
    
    # Delete multiple keys
    deleted = client.delete('user:1001', 'user:1002', 'user:1003')
    print(f"Deleted: {deleted}")  # 3
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Delete single key
    long deleted = client.del("user:1001");
    System.out.println("Deleted: " + deleted);  // 1
    
    // Delete multiple keys
    deleted = client.del("user:1001", "user:1002", "user:1003");
    System.out.println("Deleted: " + deleted);  // 3
}

KEYS - Find Keys by Pattern

Find all keys matching a glob-style pattern. Use * for any characters, ? for single character.

  • Redis CLI
  • Python
  • Java
  • C#
# All keys starting with "user:"
KEYS user:*
# Returns: ["user:1001", "user:1002", ...]

# All keys
KEYS *

# Single character wildcard
KEYS user:100?
# Matches: user:1001, user:1002, etc.
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Find all user keys
    keys = client.keys('user:*')
    print(f"User keys: {keys}")
    
    # All keys in region
    all_keys = client.keys('*')
    print(f"Total keys: {len(all_keys)}")
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Find all user keys
    Set<String> keys = client.keys("user:*");
    System.out.println("User keys: " + keys);
    
    // All keys in region
    Set<String> allKeys = client.keys("*");
    System.out.println("Total keys: " + allKeys.size());
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Find all user keys
var keys = client.Keys("user:*");
Console.WriteLine($"User keys: {string.Join(", ", keys)}");

// All keys in region
var allKeys = client.Keys("*");
Console.WriteLine($"Total keys: {allKeys.Count()}");
KEYS REGEX - Find Keys by Regular Expression v2.2.0

Find keys matching a Java regular expression pattern. More powerful than glob patterns - supports full regex syntax. REST API only (not available via Redis protocol).

Tip: Use keys_regex()/keysRegex() when you only need key names (faster). Use search_keys()/SearchKeysAsync() when you need both keys and values.
REST API Endpoints
GET /api/cache/{region}/keys/regex?pattern={regex}&limit={limit} Returns keys only (faster)
GET /api/cache/{region}/ksearch?pattern={regex}&limit={limit} Returns key, value, type, TTL
  • REST
  • Python
  • Java
  • C#
# Find keys matching regex (keys only)
GET /api/cache/default/keys/regex?pattern=^user:\d%2B$&limit=1000

# Find keys with values (full search)
GET /api/cache/default/ksearch?pattern=^order:2024.*&limit=500

# Common regex patterns:
# ^user:\d+$       - user: followed by digits
# .*@gmail\.com$   - ends with @gmail.com
# ^(order|invoice):.*  - starts with order: or invoice:
from kuber import KuberRestClient

client = KuberRestClient('localhost', 8080, api_key='kub_xxx')

# Find keys matching regex pattern (keys only - faster)
keys = client.keys_regex(r'^user:\d+$')
print(f"Found {len(keys)} user keys")

# With limit
keys = client.keys_regex(r'.*@gmail\.com$', limit=500)

# Get keys WITH values (slower but includes data)
results = client.search_keys(r'^order:2024.*')
for r in results:
    print(f"Key: {r['key']}, Value: {r['value']}, TTL: {r['ttl']}")
KuberRestClient client = new KuberRestClient("localhost", 8080, "kub_xxx");

// Find keys matching regex pattern (keys only - faster)
List<String> keys = client.keysRegex("^user:\\d+$");
System.out.println("Found " + keys.size() + " user keys");

// With limit
keys = client.keysRegex(".*@gmail\\.com$", 500);

// Get keys WITH values (slower but includes data)
List<Map<String, Object>> results = client.ksearch("^order:2024.*");
for (Map<String, Object> r : results) {
    System.out.println("Key: " + r.get("key") + ", Value: " + r.get("value"));
}
using var client = new KuberRestClient("localhost", 8080, "kub_xxx");

// Find keys matching regex pattern (keys only - faster)
var keys = await client.KeysRegexAsync(@"^user:\d+$");
Console.WriteLine($"Found {keys.Count()} user keys");

// With limit
keys = await client.KeysRegexAsync(@".*@gmail\.com$", 500);

// Get keys WITH values (slower but includes data)
var results = await client.SearchKeysAsync(@"^order:2024.*");
foreach (var r in results)
{
    Console.WriteLine($"Key: {r.Key}, Value: {r.Value}, TTL: {r.Ttl}");
}
SCAN - Iterate Keys (Cursor-based)

Incrementally iterate over keys. Better than KEYS for large datasets - doesn't block server.

  • Redis CLI
  • Python
  • Java
  • C#
# Start scan (cursor 0)
SCAN 0 MATCH user:* COUNT 100
# Returns: [next_cursor, [keys...]]

# Continue with returned cursor until cursor is 0
SCAN 42 MATCH user:* COUNT 100
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Iterate all matching keys
    cursor = 0
    all_keys = []
    while True:
        cursor, keys = client.scan(cursor, match='user:*', count=100)
        all_keys.extend(keys)
        if cursor == 0:
            break
    print(f"Found {len(all_keys)} keys")
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    List<String> allKeys = new ArrayList<>();
    String cursor = "0";
    do {
        ScanResult result = client.scan(cursor, "user:*", 100);
        allKeys.addAll(result.getKeys());
        cursor = result.getCursor();
    } while (!cursor.equals("0"));
    System.out.println("Found " + allKeys.size() + " keys");
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

var allKeys = new List<string>();
long cursor = 0;
do {
    var result = await client.ScanAsync(cursor, "user:*", 100);
    allKeys.AddRange(result.Keys);
    cursor = result.Cursor;
} while (cursor != 0);
Console.WriteLine($"Found {allKeys.Count} keys");

KSEARCH - Regex Key Search Kuber Extension

Search keys using regex patterns and return keys with their values. More powerful than KEYS.

KEYS vs KSEARCH Comparison
FeatureKEYSKSEARCH
PatternGlob (user:*)Regex (user:\d+)
ReturnsKey names onlyKeys + values + metadata
Use caseSimple patternsComplex matching
  • Redis CLI
  • Python
  • Java
  • C#
# Regex search - returns keys AND values
KSEARCH user:100[0-9] LIMIT 100
# Returns: [{"key": "user:1001", "value": "..."}, ...]

# Complex regex
KSEARCH session:[a-z0-9]{8} LIMIT 50
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Regex search with values
    results = client.ksearch('user:100[0-9]', limit=100)
    for item in results:
        print(f"Key: {item['key']}, Value: {item['value']}")
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Regex search with values
    List<KeyValue> results = client.ksearch("user:100[0-9]", 100);
    for (KeyValue kv : results) {
        System.out.println("Key: " + kv.getKey() + ", Value: " + kv.getValue());
    }
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Regex search with values
var results = await client.KSearchAsync("user:100[0-9]", 100);
foreach (var kv in results) {
    Console.WriteLine($"Key: {kv.Key}, Value: {kv.Value}");
}

SET - Set String Value

Set the string value of a key. If the key already exists, it is overwritten.

  • Redis CLI
  • Python
  • Java
  • C#
# Basic SET
SET greeting "Hello, World!"

# SET with expiration (60 seconds)
SET session:token "abc123" EX 60

# SET only if not exists (NX)
SET lock:resource "locked" NX

# SET only if exists (XX)
SET counter "100" XX
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Basic SET
    client.set('greeting', 'Hello, World!')
    
    # SET with TTL (60 seconds)
    client.set('session:token', 'abc123', ttl=60)
    
    # SET only if not exists
    result = client.setnx('lock:resource', 'locked')
    print(f"Lock acquired: {result}")  # True if set
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Basic SET
    client.set("greeting", "Hello, World!");
    
    // SET with TTL (60 seconds)
    client.setex("session:token", 60, "abc123");
    
    // SET only if not exists
    boolean acquired = client.setnx("lock:resource", "locked");
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Basic SET
await client.SetAsync("greeting", "Hello, World!");

// SET with TTL (60 seconds)
await client.SetAsync("session:token", "abc123", TimeSpan.FromSeconds(60));

// SET only if not exists
bool acquired = await client.SetNxAsync("lock:resource", "locked");

GETSET / SETNX / SETEX - Atomic Operations

Atomic operations for concurrency control and TTL management.

  • Redis CLI
  • Python
  • Java
  • C#
# GETSET - atomically set new, return old
SET counter "100"
GETSET counter "200"    # Returns: "100"
GET counter             # Returns: "200"

# SETNX - set only if not exists (for locks)
SETNX lock:job "process-1"   # Returns: 1 (success)
SETNX lock:job "process-2"   # Returns: 0 (already exists)

# SETEX - set with TTL
SETEX session:token 3600 "abc123"
# GETSET
client.set('counter', '100')
old = client.getset('counter', '200')  # Returns: "100"

# SETNX - acquire lock
acquired = client.setnx('lock:job', 'process-1')  # True

# SETEX - set with TTL
client.setex('session:token', 3600, 'abc123')
// GETSET
client.set("counter", "100");
String old = client.getSet("counter", "200");  // "100"

// SETNX - acquire lock
boolean acquired = client.setnx("lock:job", "process-1");

// SETEX - set with TTL
client.setex("session:token", 3600, "abc123");
// GETSET
await client.SetAsync("counter", "100");
string? old = await client.GetSetAsync("counter", "200");

// SETNX - acquire lock
bool acquired = await client.SetNxAsync("lock:job", "process-1");

// SETEX - set with TTL
await client.SetExAsync("session:token", 3600, "abc123");

MGET - Get Multiple Keys

Get the values of multiple keys in a single request. Returns nil for keys that don't exist.

  • Redis CLI
  • Python
  • Java
  • C#
# Set some keys first
SET user:1001 "Alice"
SET user:1002 "Bob"
SET user:1003 "Charlie"

# Get multiple keys
MGET user:1001 user:1002 user:1003 user:9999
# Returns: ["Alice", "Bob", "Charlie", nil]

# Order is preserved - nil for missing keys
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Set some keys
    client.set('user:1001', 'Alice')
    client.set('user:1002', 'Bob')
    client.set('user:1003', 'Charlie')
    
    # Get multiple keys at once
    values = client.mget('user:1001', 'user:1002', 'user:1003', 'user:9999')
    print(f"Values: {values}")
    # ['Alice', 'Bob', 'Charlie', None]
    
    # Or use list
    keys = ['user:1001', 'user:1002', 'user:1003']
    values = client.mget(*keys)
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Set some keys
    client.set("user:1001", "Alice");
    client.set("user:1002", "Bob");
    client.set("user:1003", "Charlie");
    
    // Get multiple keys at once
    List<String> values = client.mget("user:1001", "user:1002", "user:1003", "user:9999");
    System.out.println("Values: " + values);
    // [Alice, Bob, Charlie, null]
    
    // Or use List
    List<String> keys = Arrays.asList("user:1001", "user:1002", "user:1003");
    values = client.mget(keys.toArray(new String[0]));
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Set some keys

// Alice, Bob, Charlie, null

MSETNX - Set Multiple Keys If None Exist

Set multiple keys only if none of them exist. Atomic all-or-nothing operation.

  • Redis CLI
  • Python
  • Java
  • C#
# First call - all keys are new
MSETNX newkey1 "value1" newkey2 "value2" newkey3 "value3"
# Returns: 1 (success - all keys set)

# Second call - some keys exist
MSETNX newkey1 "updated" newkey4 "value4"
# Returns: 0 (failed - newkey1 exists, NONE set)

# newkey4 was NOT set because newkey1 existed
GET newkey4
# Returns: (nil)
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # First call - success
    result = client.msetnx({
        'newkey1': 'value1',
        'newkey2': 'value2',
        'newkey3': 'value3'
    })
    print(f"First MSETNX: {result}")  # True
    
    # Second call - fails (newkey1 exists)
    result = client.msetnx({
        'newkey1': 'updated',
        'newkey4': 'value4'
    })
    print(f"Second MSETNX: {result}")  # False (nothing set)
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // First call - success
    Map<String, String> first = Map.of(
        "newkey1", "value1",
        "newkey2", "value2",
        "newkey3", "value3"
    );
    boolean result = client.msetnx(first);
    System.out.println("First MSETNX: " + result);  // true
    
    // Second call - fails (newkey1 exists)
    Map<String, String> second = Map.of(
        "newkey1", "updated",
        "newkey4", "value4"
    );
    result = client.msetnx(second);
    System.out.println("Second MSETNX: " + result);  // false
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// First call - success
var first = new Dictionary<string, string> {
    ["newkey1"] = "value1",
    ["newkey2"] = "value2",
    ["newkey3"] = "value3"
};
bool result = await client.MSetNxAsync(first);
Console.WriteLine($"First MSETNX: {result}");  // true

// Second call - fails (newkey1 exists)
var second = new Dictionary<string, string> {
    ["newkey1"] = "updated",
    ["newkey4"] = "value4"
};
result = await client.MSetNxAsync(second);

DEL - Delete Multiple Keys

Delete multiple keys in a single operation. Returns the number of keys deleted.

  • Redis CLI
  • Python
  • Java
  • C#
# Delete multiple keys
DEL user:1001 user:1002 user:1003 nonexistent
# Returns: 3 (only existing keys counted)

# Useful pattern: delete by pattern (two-step)
# 1. Get keys
KEYS session:*
# 2. Delete them
DEL session:abc session:def session:ghi
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:

    # Delete by pattern (two-step)
    keys = client.keys('session:*')
    if keys:
        deleted = client.delete(*keys)
        print(f"Deleted {deleted} session keys")

Performance Tips
  • Use batch operations instead of multiple single calls to reduce network round-trips
  • MGET/MSET are significantly faster than looping over individual GET/SET calls
  • Optimal batch size: 100-1000 keys per batch for best performance
  • Messaging limits: When using request/response messaging, batch operations are limited by max_batch_get_size and max_batch_set_size (default: 128)
Example: Batch Loading
# Python - efficiently load 10,000 keys
def batch_load(client, key_values, batch_size=500):
    items = list(key_values.items())
    for i in range(0, len(items), batch_size):
        batch = dict(items[i:i+batch_size])
        client.mset(batch)
        print(f"Loaded {min(i+batch_size, len(items))}/{len(items)}")
