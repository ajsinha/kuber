Key Operations

Key operations allow you to manage cache keys - check existence, delete, rename, find keys by pattern, and more.

Command Reference
CommandSyntaxDescription
DELDEL key [key ...]Delete one or more keys
EXISTSEXISTS key [key ...]Check if key(s) exist
KEYSKEYS patternFind keys matching glob pattern
KEYS/REGEXGET /api/cache/{region}/keys/regexFind keys matching regex (REST only, v2.2.0)
SCANSCAN cursor [MATCH pattern] [COUNT count]Incrementally iterate keys
TYPETYPE keyGet type of value at key
RENAMERENAME key newkeyRename a key
RENAMENXRENAMENX key newkeyRename only if newkey doesn't exist
DBSIZEDBSIZEGet number of keys in current region
FLUSHDBFLUSHDBDelete all keys in current region
KSEARCHGET /api/cache/{region}/ksearchRegex key search with values (REST only)
DEL - Delete Keys

Delete one or more keys. Returns the number of keys deleted.

  • Redis CLI
  • Python
  • Java
  • C#
# Delete single key
DEL user:1001
# Returns: 1 (if existed)

# Delete multiple keys
DEL user:1001 user:1002 user:1003
# Returns: 3 (number deleted)

# Delete non-existent key
DEL nonexistent
# Returns: 0
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Delete single key
    deleted = client.delete('user:1001')
    print(f"Deleted: {deleted}")  # 1
    
    # Delete multiple keys
    deleted = client.delete('user:1001', 'user:1002', 'user:1003')
    print(f"Deleted: {deleted}")  # 3
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Delete single key
    long deleted = client.del("user:1001");
    System.out.println("Deleted: " + deleted);  // 1
    
    // Delete multiple keys
    deleted = client.del("user:1001", "user:1002", "user:1003");
    System.out.println("Deleted: " + deleted);  // 3
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Delete single key
long deleted = await client.DeleteAsync("user:1001");
Console.WriteLine($"Deleted: {deleted}");  // 1

// Delete multiple keys
deleted = await client.DeleteAsync("user:1001", "user:1002", "user:1003");
Console.WriteLine($"Deleted: {deleted}");  // 3
EXISTS - Check Key Existence

Check if one or more keys exist. Returns count of existing keys.

  • Redis CLI
  • Python
  • Java
  • C#
# Check single key
EXISTS user:1001
# Returns: 1 (exists) or 0 (not exists)

# Check multiple keys
EXISTS user:1001 user:1002 user:1003
# Returns: count of existing keys (0-3)
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Check single key
    exists = client.exists('user:1001')
    print(f"Exists: {exists}")  # True/False
    
    # Check multiple keys
    count = client.exists('user:1001', 'user:1002', 'user:1003')
    print(f"Existing count: {count}")  # 0-3
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Check single key
    boolean exists = client.exists("user:1001");
    System.out.println("Exists: " + exists);
    
    // Check multiple keys
    long count = client.exists("user:1001", "user:1002", "user:1003");
    System.out.println("Existing count: " + count);
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Check single key
bool exists = await client.ExistsAsync("user:1001");
Console.WriteLine($"Exists: {exists}");

// Check multiple keys
long count = await client.ExistsAsync("user:1001", "user:1002", "user:1003");
Console.WriteLine($"Existing count: {count}");
KEYS - Find Keys by Pattern

Find all keys matching a glob-style pattern. Use * for any characters, ? for single character.

  • Redis CLI
  • Python
  • Java
  • C#
# All keys starting with "user:"
KEYS user:*
# Returns: ["user:1001", "user:1002", ...]

# All keys
KEYS *

# Single character wildcard
KEYS user:100?
# Matches: user:1001, user:1002, etc.
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Find all user keys
    keys = client.keys('user:*')
    print(f"User keys: {keys}")
    
    # All keys in region
    all_keys = client.keys('*')
    print(f"Total keys: {len(all_keys)}")
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Find all user keys
    Set<String> keys = client.keys("user:*");
    System.out.println("User keys: " + keys);
    
    // All keys in region
    Set<String> allKeys = client.keys("*");
    System.out.println("Total keys: " + allKeys.size());
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Find all user keys
var keys = client.Keys("user:*");
Console.WriteLine($"User keys: {string.Join(", ", keys)}");

// All keys in region
var allKeys = client.Keys("*");
Console.WriteLine($"Total keys: {allKeys.Count()}");
KEYS REGEX - Find Keys by Regular Expression v2.2.0

Find keys matching a Java regular expression pattern. More powerful than glob patterns - supports full regex syntax. REST API only (not available via Redis protocol).

Tip: Use keys_regex()/keysRegex() when you only need key names (faster). Use search_keys()/SearchKeysAsync() when you need both keys and values.
REST API Endpoints
GET /api/cache/{region}/keys/regex?pattern={regex}&limit={limit} Returns keys only (faster)
GET /api/cache/{region}/ksearch?pattern={regex}&limit={limit} Returns key, value, type, TTL
  • REST
  • Python
  • Java
  • C#
# Find keys matching regex (keys only)
GET /api/cache/default/keys/regex?pattern=^user:\d%2B$&limit=1000

# Find keys with values (full search)
GET /api/cache/default/ksearch?pattern=^order:2024.*&limit=500

# Common regex patterns:
# ^user:\d+$       - user: followed by digits
# .*@gmail\.com$   - ends with @gmail.com
# ^(order|invoice):.*  - starts with order: or invoice:
from kuber import KuberRestClient

client = KuberRestClient('localhost', 8080, api_key='kub_xxx')

# Find keys matching regex pattern (keys only - faster)
keys = client.keys_regex(r'^user:\d+$')
print(f"Found {len(keys)} user keys")

# With limit
keys = client.keys_regex(r'.*@gmail\.com$', limit=500)

# Get keys WITH values (slower but includes data)
results = client.search_keys(r'^order:2024.*')
for r in results:
    print(f"Key: {r['key']}, Value: {r['value']}, TTL: {r['ttl']}")
KuberRestClient client = new KuberRestClient("localhost", 8080, "kub_xxx");

// Find keys matching regex pattern (keys only - faster)
List<String> keys = client.keysRegex("^user:\\d+$");
System.out.println("Found " + keys.size() + " user keys");

// With limit
keys = client.keysRegex(".*@gmail\\.com$", 500);

// Get keys WITH values (slower but includes data)
List<Map<String, Object>> results = client.ksearch("^order:2024.*");
for (Map<String, Object> r : results) {
    System.out.println("Key: " + r.get("key") + ", Value: " + r.get("value"));
}
using var client = new KuberRestClient("localhost", 8080, "kub_xxx");

// Find keys matching regex pattern (keys only - faster)
var keys = await client.KeysRegexAsync(@"^user:\d+$");
Console.WriteLine($"Found {keys.Count()} user keys");

// With limit
keys = await client.KeysRegexAsync(@".*@gmail\.com$", 500);

// Get keys WITH values (slower but includes data)
var results = await client.SearchKeysAsync(@"^order:2024.*");
foreach (var r in results)
{
    Console.WriteLine($"Key: {r.Key}, Value: {r.Value}, TTL: {r.Ttl}");
}
SCAN - Iterate Keys (Cursor-based)

Incrementally iterate over keys. Better than KEYS for large datasets - doesn't block server.

  • Redis CLI
  • Python
  • Java
  • C#
# Start scan (cursor 0)
SCAN 0 MATCH user:* COUNT 100
# Returns: [next_cursor, [keys...]]

# Continue with returned cursor until cursor is 0
SCAN 42 MATCH user:* COUNT 100
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Iterate all matching keys
    cursor = 0
    all_keys = []
    while True:
        cursor, keys = client.scan(cursor, match='user:*', count=100)
        all_keys.extend(keys)
        if cursor == 0:
            break
    print(f"Found {len(all_keys)} keys")
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    List<String> allKeys = new ArrayList<>();
    String cursor = "0";
    do {
        ScanResult result = client.scan(cursor, "user:*", 100);
        allKeys.addAll(result.getKeys());
        cursor = result.getCursor();
    } while (!cursor.equals("0"));
    System.out.println("Found " + allKeys.size() + " keys");
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

var allKeys = new List<string>();
long cursor = 0;
do {
    var result = await client.ScanAsync(cursor, "user:*", 100);
    allKeys.AddRange(result.Keys);
    cursor = result.Cursor;
} while (cursor != 0);
Console.WriteLine($"Found {allKeys.Count} keys");
TYPE / RENAME - Key Information & Management

Get the type of value at a key and rename keys.

  • Redis CLI
  • Python
  • Java
  • C#
# Get type of value
TYPE mykey
# Returns: string, hash, list, set, zset, or none

# Rename key
RENAME oldkey newkey

# Rename only if newkey doesn't exist
RENAMENX oldkey newkey
# Returns: 1 (success) or 0 (newkey exists)
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Get type
    key_type = client.type('mykey')
    print(f"Type: {key_type}")  # "string", "hash", etc.
    
    # Rename key
    client.rename('oldkey', 'newkey')
    
    # Rename only if newkey doesn't exist
    success = client.renamenx('oldkey', 'newkey')
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Get type
    String keyType = client.type("mykey");
    System.out.println("Type: " + keyType);
    
    // Rename key
    client.rename("oldkey", "newkey");
    
    // Rename only if newkey doesn't exist
    boolean success = client.renamenx("oldkey", "newkey");
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Get type
string keyType = await client.TypeAsync("mykey");
Console.WriteLine($"Type: {keyType}");

// Rename key
await client.RenameAsync("oldkey", "newkey");

// Rename only if newkey doesn't exist
bool success = await client.RenameNxAsync("oldkey", "newkey");
KSEARCH - Regex Key Search Kuber Extension

Search keys using regex patterns and return keys with their values. More powerful than KEYS.

  • Redis CLI
  • Python
  • Java
  • C#
# Regex search - returns keys AND values
KSEARCH user:100[0-9] LIMIT 100
# Returns: [{"key": "user:1001", "value": "..."}, ...]

# Complex regex
KSEARCH session:[a-z0-9]{8} LIMIT 50
with KuberRedisClient('localhost', 6380, api_key='kub_xxx') as client:
    # Regex search with values
    results = client.ksearch('user:100[0-9]', limit=100)
    for item in results:
        print(f"Key: {item['key']}, Value: {item['value']}")
try (KuberClient client = new KuberClient("localhost", 6380, "kub_xxx")) {
    // Regex search with values
    List<KeyValue> results = client.ksearch("user:100[0-9]", 100);
    for (KeyValue kv : results) {
        System.out.println("Key: " + kv.getKey() + ", Value: " + kv.getValue());
    }
}
using var client = new KuberClient("localhost", 6380, "kub_xxx");

// Regex search with values
var results = await client.KSearchAsync("user:100[0-9]", 100);
foreach (var kv in results) {
    Console.WriteLine($"Key: {kv.Key}, Value: {kv.Value}");
}
