Region Management

Overview

Regions are logical namespaces for organizing cache data. Regions are auto-created when data is stored.

Region Isolation (v1.2.2): Each region gets its own dedicated database instance (RocksDB or SQLite). This provides better concurrency, isolation, and the ability to compact individual regions independently.

Region Name Rules

Region names must follow these rules:

  • Only alphanumeric characters and underscores: [a-zA-Z0-9_]
  • Must contain at least one alphabetic character
  • Valid: default, customers, cache_v2, Region1
  • Invalid: 123, _, my-region, cache.data

Per-Region Memory Limits (v1.2.2)

Configure memory limits for individual regions:

# application.yml
kuber:
  cache:
    max-memory-entries: 100000           # Default for all regions
    global-max-memory-entries: 500000    # Global cap (0 = unlimited)
    region-memory-limits:
      customers: 50000                   # Override for customers region
      products: 200000                   # Override for products region

REST API

MethodEndpointDescription
GET/api/regionsList all regions
POST/api/regionsCreate region
GET/api/regions/{name}Get region info
DELETE/api/regions/{name}Delete region
POST/api/regions/{name}/purgeClear all entries

Attribute Mapping

Transform JSON field names when storing data:

PUT /api/regions/customers/attributemapping
{"firstName": "first_name", "lastName": "last_name"}
