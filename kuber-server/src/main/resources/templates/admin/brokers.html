Glossary & Reference
Broker Types
Apache Kafka
Distributed event streaming platform. Uses bootstrap servers for initial cluster discovery. Best for high-throughput, ordered event streams.
Confluent Kafka
Confluent Cloud or Platform. Uses API key/secret with SASL_SSL automatically — no manual SSL setup. Set type to confluent-kafka.
ActiveMQ
JMS-compliant message broker. Uses broker URL (e.g., tcp://host:61616). Best for traditional point-to-point and pub/sub messaging.
RabbitMQ
AMQP message broker. Uses host/port with optional virtual host. Best for flexible routing, dead-letter queues, and priority messaging.
IBM MQ
Enterprise messaging. Uses queue manager + channel + conn_name. Best for mainframe integration and regulated industries.
File
Writes events to local disk in JSONL format. Uses a directory path. Best for development, audit trails, and environments without a message broker.
SSL/TLS Modes
JKS / PKCS12
Java KeyStore trust store for server certificate verification. Provide .jks or .p12 file + password.
PEM
PEM CA certificate for server verification. Provide a .pem or .crt file containing the CA certificate chain.
SASL_SSL (Kafka)
Kafka-specific: combines SSL transport with SASL authentication (PLAIN, SCRAM, OAUTHBEARER). Requires JAAS config.
mTLS JKS
Mutual TLS with Java KeyStores. Both client and server authenticate. Requires key store (client cert) + trust store (server CA).
mTLS PEM
Mutual TLS with PEM files. Client cert + key + CA cert. Common in Kubernetes and cloud environments.
Concepts
Enabled
Broker connection is active. Disabled brokers are defined in config but not connected — no resources consumed.
JSON Editor
Edit the raw message_brokers.json file directly. Advanced users can paste entire broker configs. Validates JSON before saving.
Config File
Brokers are stored in config/message_brokers.json (configurable via kuber.publishing.broker-config-file). Changes are hot-reloaded.
Example Config
{
  "production-kafka": {
    "type": "kafka",
    "enabled": true,
    "bootstrap_servers": "kafka1:9092"
  },
  "confluent-cloud": {
    "type": "confluent-kafka",
    "enabled": true,
    "bootstrap-servers": "pkc-xxx.cloud:9092",
    "api-key": "KEY", "api-secret": "SECRET"
  }
}
